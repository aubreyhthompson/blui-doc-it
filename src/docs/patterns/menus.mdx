# Basic Menu

<ImageGrid
    images={[SimpleDesktopMenu]}
    regularWidth
    caption={`A simple dropdown menu accessed from the 'View More' button.`}
/>

The basic menu presents actions or options on a surface that appears when a user interacts with a UI element. A familiar example is the File or Edit menu usually present at the top of an application.

Menus can contain a flat list of actions, or can be hierarchical where some menu items open subsequent menus with further options. When selecting a menu item, the action should be taken and the menu is closed.

# User Menu

<ImageGrid
    images={[DesktopUserMenu]}
    regularWidth
    caption={`The user menu is controlled by an Avatar button and displays useful user-related information and actions.`}
/>

The User Menu is a special case of the menu, which can be accessed once a user has logged in to an application. It appears as an Avatar in the [App Bar](/patterns/appbar), usually displaying the user's initials or a user-selected profile picture. The associated menu typically contains user-specific content, such as links to the user profile / settings, log out functionality, etc. Further, the menu should also display basic information about the user when opened, such as their name and / or username.

# Behaviors

## Menu vs. Bottom Sheet

In general, you should stick to using menus only for desktop applications. Mobile applications (and web applications viewed on mobile-sized devices) should follow the mobile convention of using a [Bottom Sheet](/patterns/overlay#bottom-sheet) instead.
